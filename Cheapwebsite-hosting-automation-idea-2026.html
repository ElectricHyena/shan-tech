Executive Summary

This document outlines a fully automated website business model that leverages AI and cloud infrastructure to deliver professional websites at £4.99/month with 94% profit margins.

The Core Insight: Platforms like Vercel are just wrappers around AWS/GCP. By going directly to cloud infrastructure (specifically Cloudflare R2), hosting costs drop to near-zero, enabling a sustainable £4.99/month price point that undercuts everyone while maintaining healthy margins.

Where Does Kubernetes Fit?

Use Kubernetes For:

  • n8n automation platform
  • Customer dashboard/portal
  • API services
  • Preview environments
  • Build queue workers

Don't Use Kubernetes For:

  • Serving static files (use R2/S3)
  • CDN functionality (use Cloudflare/CloudFront)
  • SSL termination (handled by CDN)
  • DNS (use Cloudflare/Route53)

Hybrid Architecture with K8s

K8s Cluster (EKS/GKE) ~£100-150/mo: ├── n8n (automation workflows) ├── API service (custom logic) ├── Build workers (generate sites) └── Customer dashboard (portal) External (NOT in K8s): ├── Cloudflare R2 (static file storage) ├── Cloudflare CDN (global delivery) ├── Cloudflare DNS (domain management) └── Stripe (billing)
Verdict: For MVP, skip Kubernetes. A single EC2/VPS with Docker Compose is sufficient. Add K8s later when you need the customer dashboard, complex workflows, or want autoscaling.
